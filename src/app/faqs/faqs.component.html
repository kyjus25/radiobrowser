I count only one click per ip aliress per station a day. Some people were clicking their favourite station 100s of times every day by using the programmers API.

It depends on the overall number of stations in the database and if the station is currently marked as broken. Currently a cron job is run every 10 minutes which checks 50 working stations and 10 broken stations.

The stations that were not checked for the longest time are checked first.

This can have multiple causes:

  • Legal restrictions:
    • There may be some countries which have legal restrictions prohibiting your favourite station from broadcasting worldwide. It has to be possible to connect to it from my server which is located in Germany.
  • High Costs:
    • Some others would allow global broadcast but let the stations pay huge amounts of money to copyright owners if they want to do that. Please talk to your station or your local political representative to ensure the freedom of global broadcasting.
  • Status not up to date:
    • Sometimes streams go down for some time. If the automatic check happens to be there at this exact time, they will be marked as broken. It will take some time that this station is checked again and marked as usable.
  • Bugs:
    • My program has a BUG :) If you think that is the case, please drop me a note or if you are a programmer, fix it on github.

There are 2 cases in which stations will get deleted automatically:

  • New stations that never worked will get deleted after 3 days.
  • Stations that did work at some point, but have not been working for at least 30 days now will get deleted.

Does radio-browser have some kind of federation network?

The new api radiobrowser-api-rust does have federation.
The old one radiobrowser-api does not.

What is the future plan for www.radio-browser.info?

Current State
At the moment www.radio-browser.info/webservice is the only first level server hanping most users.
There are some second level servers which already use the new style api and use federation to mirror the content of the first level server.
For now stations are distributed from the first level server to the second level servers and NOT in the other direction. That means stations added to second level servers will not get distributed and will be deleted eventually.

Future
The server at www.radio-browser.info/webservice will be shut down eventually and the current second level servers will become the first level servers for other people to use and mirror.
Stations can be aped to any one of the new first level servers and will be distributed among them.

How does federation work?

Every server does keep a station change history which does keep all apitions and changes. This can be pulled from other servers as a full version or as a differential from the state they already have. Each server also does keep a history of station checks, which can also be pulled in a similiar fashion.

radiobrowser-api-rust

Language: RUST
Provides API for multiple second level servers at

  • de1.api.radio-browser.info
  • de2.api.radio-browser.info
  • nl1.api.radio-browser.info

http-distribute-requests

Language: NodeJs
Acts as a central point for handing out redirects to the other second level servers
Available at: api.radio-browser.info (this only gives out redirects to the others)

  • radiobrowser-api
    • Language: PHP
      DEPRECATED, will only live until api.radio-browser.info has all features.
      Provides the API at http://www.radio-browser.info/webservice/.
  • radiobrowser
    • Language: HTML, Javascript, Angular 1
      Basic web page to list, play and ali stations. Communicate to radiobrowser-api through REST requests.
  • stream-check
    • Language: Rust
      Depends on: av-stream-info
      DEPRECATED, will only live until api.radio-browser.info has all features.
      Provides stream check at www.radio-browser.info
      Check streams every 24 hours for metadata and if they are still online. Save the data to the database.
      Remove broken stations after some time.
  • radiobrowser-cleanup-tasks
    • Language: Rust
      DEPRECATED, will only live until api.radio-browser.info has all features.
      Provides database cleanup at www.radio-browser.info
      Refresh the caches.
  • av-stream-check
    • Language: Rust
      Type: Library
      Repository: crates.io
      Depends on: playlist-decoder
      Library that checks audio and video streams and extracts metadata.
  • playlist-decoder
    • Language: Rust
      Type: Library
      Repository: crates.io
      Library that decodes various playlist formats (M3U,PLS,XSPF,)
