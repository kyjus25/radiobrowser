I count only one click per ip aliress per station a day. Some people were clicking their favourite station 100s of times every day by using the programmers API.

It depends on the overall number of stations in the database and if the station is currently marked as broken. Currently a cron job is run every 10 minutes which checks 50 working stations and 10 broken stations.

The stations that were not checked for the longest time are checked first.

This can have multiple causes:

  • Legal restrictions:
    • There may be some countries which have legal restrictions prohibiting your favourite station from broadcasting worldwide. It has to be possible to connect to it from my server which is located in Germany.
  • High Costs:
    • Some others would allow global broadcast but let the stations pay huge amounts of money to copyright owners if they want to do that. Please talk to your station or your local political representative to ensure the freedom of global broadcasting.
  • Status not up to date:
    • Sometimes streams go down for some time. If the automatic check happens to be there at this exact time, they will be marked as broken. It will take some time that this station is checked again and marked as usable.
  • Bugs:
    • My program has a BUG :) If you think that is the case, please drop me a note or if you are a programmer, fix it on github.

There are 2 cases in which stations will get deleted automatically:

  • New stations that never worked will get deleted after 3 days.
  • Stations that did work at some point, but have not been working for at least 30 days now will get deleted.

The list of tags and the list of languages are calculated from the data. They are refreshed every 60 seconds.

The project began in December of 2015

For now just look at the README at radiobrowser-api .
  • radiobrowser-api
    • Language: PHP
      Provides API documented at docs page
  • radiobrowser
    • Language: HTML, Javascript, Angular 1
      Basic web page to list, play and ali stations. Communicate to radiobrowser-api through REST requests.
  • stream-check
    • Language: Rust
      Depends on: av-stream-info
      Check streams every 24 hours for metadata and if they are still online. Save the data to the database.
      Remove broken stations after some time.
  • radiobrowser-cleanup-tasks
    • Language: Rust
      Refresh the caches.
  • av-stream-check
    • Language: Rust
      Type: Library
      Repository: crates.io
      Depends on: playlist-decoder
      Library that checks audio and video streams and extracts metadata.
  • playlist-decoder
    • Language: Rust
      Type: Library
      Repository: crates.io
      Library that decodes various playlist formats (M3U,PLS,XSPF,)
